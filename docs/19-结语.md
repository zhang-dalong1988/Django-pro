# 第 19 章：结语

构建一个"专业级"网站绝非易事，即使有 Django 这样功能齐全的 Web 框架提供帮助也是如此。Docker 在标准化本地和生产环境方面提供了重大优势，无论本地机器配置如何——特别是在团队协作环境中。然而，Docker 本身就是一个复杂的工具。虽然我们在本书中谨慎地使用了它，但根据项目需求，它还能做更多的事情。

Django 本身对小型项目很友好，因为它的默认设置强调快速的本地开发，但这些设置必须为生产环境进行系统性更新——从将数据库升级到 PostgreSQL，到使用自定义用户模型、环境变量、配置用户注册流程、静态资源、邮件...等等，不一而足。

好消息是，生产级方法所需的步骤非常相似。因此，本书的前半部分故意对最终构建的项目保持不可知论：你会发现这些步骤在几乎任何新的 Django 项目中都是标准的。后半部分专注于使用现代最佳实践构建真实的书店网站，添加了评论功能、图片上传、设置权限、添加搜索、审查性能和安全措施，最后使用容器在 Heroku 上部署。

尽管本书涵盖了所有这些内容，但我们实际上只是触及了 Django 能力的表面。这就是现代 Web 开发的本质：持续迭代。

Django 是构建专业网站的绝佳伙伴，因为许多必需的考虑因素都已经被考虑并包含在内。但需要知识来了解如何开启这些生产开关，以充分利用 Django 允许的定制化。最终，这就是本书的目标：让你，读者，接触 Django 和专业网站所需的全方位内容。

在你学习更多关于 Web 开发和 Django 的知识时，我建议在过早优化方面要谨慎。总是很容易向你的项目添加你认为以后会需要的功能和优化。简短的清单包括为静态和媒体资源添加 CDN、明智地分析数据库查询、为模型添加索引等等。

事实是，在任何给定的 Web 项目中，总是有比时间允许的更多工作要做。本书涵盖了值得前期投入时间做对的基础知识。围绕安全性、性能和功能的其他步骤将实时呈现给你。尽量抵制添加复杂性的冲动，直到绝对必要。
